# Comprehensive macOS Storage Paths Reference

This is the comprehensive path reference that all scanner agents use when identifying cleanable storage.

## 1. System Caches & Logs (SAFE)

**User Caches:**
- `~/Library/Caches/*` — Per-application caches, auto-regenerated by the system
- `/tmp/*` — Temporary files created by applications and the OS
- `/private/tmp/*` — Private temporary files
- `/var/folders/*/*/C/` — Per-user temporary cache directory
- `~/Library/Saved Application State/` — Application state files (safe to delete, apps will regenerate)

**System Logs:**
- `~/Library/Logs/*` — Application log files (safe if >30 days old)
- `/Library/Logs/DiagnosticReports/` — Crash reports and diagnostic logs
- `/var/log/*` — System log directory (requires elevated privileges)

**Application Metadata:**
- `~/Library/Metadata/` — Application metadata (safe to delete)

## 2. Browser Caches (SAFE)

**Chrome/Chromium-based:**
- `~/Library/Caches/Google/Chrome/` — Main cache directory
- `~/Library/Application Support/Google/Chrome/Default/Service Worker/CacheStorage/` — Service worker cache
- `~/Library/Application Support/Google/Chrome/Default/Cache/` — Additional cache
- `~/Library/Application Support/Google/Chrome/Default/Code Cache/` — V8 code cache

**Firefox:**
- `~/Library/Caches/Firefox/Profiles/` — Firefox profile cache
- `~/Library/Application Support/Firefox/Profiles/*/cache2/` — Cache storage

**Safari:**
- `~/Library/Caches/com.apple.Safari/` — Safari cache
- `~/Library/Safari/History.db-wal` — History database temporary files
- `~/Library/Safari/TopSites.plist` — Top sites cache

**Edge:**
- `~/Library/Caches/Microsoft Edge/` — Edge cache directory
- `~/Library/Application Support/Microsoft Edge/Default/Cache/` — Additional cache

**Brave:**
- `~/Library/Caches/BraveSoftware/` — Brave cache directory
- `~/Library/Application Support/BraveSoftware/Brave-Browser/Default/Cache/`

**Arc Browser:**
- `~/Library/Caches/company.thebrowser.Browser/` — Arc cache directory
- `~/Library/Application Support/company.thebrowser.Browser/Default/Cache/`

## 3. Cloud Storage Caches (SAFE)

**iCloud Drive:**
- `~/Library/Mobile Documents/` — Local cache copies of iCloud documents
- `~/Library/Mobile Documents/com~apple~CloudDocs/` — Main iCloud Drive cache

**Dropbox:**
- `~/.dropbox.cache/` — Dropbox cache directory

**Google Drive:**
- `~/Library/Application Support/Google/DriveFS/` — Google Drive File Stream cache

**OneDrive:**
- `~/Library/CloudStorage/OneDrive*/` — OneDrive local cache (multiple entries for business/personal)

**Box.com:**
- `~/Library/Application Support/Box/` — Box cloud storage cache

**Sync.com:**
- `~/Library/Application Support/Sync.com/` — Sync.com cache

## 4. Downloads & User Files (REVIEW)

**Downloads Directory:**
- `~/Downloads/` — User downloads folder
- Flag files: `.dmg`, `.pkg`, `.zip`, `.iso`, `.tar.gz`, `.rar` files >30 days old
- Large command: `du -sh ~/Downloads/*` to show individual file sizes
- Find old files: `find ~/Downloads -mtime +30 -type f`

**Desktop:**
- `~/Desktop/` — User desktop files
- Flag: `Screenshot*.png` files (screenshot clutter)
- Flag: Old `.dmg` and installer files
- Find old files: `find ~/Desktop -mtime +30 -type f`

**Documents:**
- `~/Documents/` — User documents folder
- Flag: Large files >1GB untouched >6 months
- Search command: `mdfind -onlyin ~/Documents "kMDItemFSSize > 1073741824"`
- Find old files: `find ~/Documents -mtime +180 -size +1G -type f`

**Pictures & Movies:**
- `~/Pictures/` — Photo library (usually linked to Photos.app)
- `~/Movies/` — Video files and creative tool renders
- Xcode render files: `~/Movies/` for Final Cut Pro, Premiere Pro, etc.

## 5. Communications (REVIEW)

**Mail Application:**
- `~/Library/Mail/` — Mail message database and attachments (REQUIRES FULL DISK ACCESS)
- `~/Library/Containers/com.apple.mail/` — Sandboxed Mail container
- `~/Library/Containers/com.apple.mail.shared/` — Shared Mail data
- Warning: Full Disk Access (FDA) required; cannot be recovered after deletion

**iMessage:**
- `~/Library/Messages/` — iMessage message database (REQUIRES FULL DISK ACCESS)
- `~/Library/Messages/Attachments/` — iMessage media files (photos, videos, documents)
- `~/Library/Containers/com.apple.iChat/` — iChat/Messages container
- **Critical Warning:** iMessage media CANNOT BE RECOVERED once deleted from Trash

**Slack & Other Chat Apps:**
- `~/Library/Containers/com.tinyspeck.slackmacgap/` — Slack cache and data
- `~/Library/Application Support/Slack/` — Slack application support
- `~/Library/Caches/com.tinyspeck.slackmacgap/` — Slack cache

**Telegram:**
- `~/Library/Application Support/Telegram Desktop/` — Telegram data and cache

**Discord:**
- `~/Library/Application Support/discord/` — Discord cache and data

## 6. iOS Device Backups (REVIEW)

**Backup Location:**
- `~/Library/Application Support/MobileSync/Backup/` — All device backups

**Device Information:**
- Parse `Info.plist` from each backup folder to show device name and date
- Command: `plutil -p "~/Library/Application Support/MobileSync/Backup/*/Info.plist"`
- Extract device name with: `defaults read "~/Library/Application Support/MobileSync/Backup/<ID>/Info.plist" DeviceName`
- Extract date with: `stat -f "%Sm" "~/Library/Application Support/MobileSync/Backup/<ID>/Info.plist"`

**Size Information:**
- Calculate backup size: `du -sh "~/Library/Application Support/MobileSync/Backup/<ID>/"`
- Typical sizes: 50MB to 100GB+ depending on device

**Deletion Handling:**
- Show device name, model, and backup date before deletion
- Example: "iPhone 15 Pro, last backed up 2026-02-15, 23.4 GB"
- Recommended: Suggest user back up their device first

## 7. Time Machine (REVIEW)

**Local Snapshots:**
- List snapshots: `tmutil listlocalsnapshots /`
- Date format: YYYY-MM-DD-HHMMSS
- Size per snapshot: Typical 2-5GB each

**Snapshot Deletion:**
- Delete specific: `sudo tmutil deletelocalsnapshots <date>` (requires sudo)
- Delete all: `sudo tmutil deletelocalsnapshots /`
- **Warning:** Requires elevated privileges and password entry

**Size Estimation:**
- Count snapshots: `tmutil listlocalsnapshots / | wc -l`
- Estimate total: (count × 3.5GB typical)
- Note: Full snapshots and incremental snapshots vary in size

**Time Machine Data:**
- `~/Library/Application Support/MobileSync/Backup/` — Separate from Time Machine
- `/.Trashes/` — Trash directory (system-protected)

## 8. App Leftovers (REVIEW)

**Application Support:**
- `~/Library/Application Support/` — Per-app data for all installed applications
- Cross-reference against `/Applications/` and `~/Applications/` for uninstalled apps
- Large directories: Adobe, Blender, Xcode, Docker, Virtualbox data

**Sandboxed App Data:**
- `~/Library/Containers/` — Sandboxed application data (one folder per app)
- Format: Folder named `com.company.appname`
- Safe to delete only if app is uninstalled

**Group Containers:**
- `~/Library/Group Containers/` — Shared data between related applications
- Format: Folder named `group.com.company.appname`
- May be used by multiple apps; delete only if all related apps are uninstalled

**Preferences:**
- `~/Library/Preferences/` — Application preference files (.plist files)
- Uninstalled app prefs: `com.company.appname.plist` for apps no longer installed
- Safe to delete plist for uninstalled apps only

**LaunchAgents:**
- `~/Library/LaunchAgents/` — User-scheduled tasks (may include startup items for uninstalled apps)
- Safe to delete only if associated app is uninstalled

**LaunchDaemons:**
- `/Library/LaunchDaemons/` — System-scheduled tasks (requires elevated privileges)
- Skip unless you are certain

## 9. Creative Tools (REVIEW)

**Adobe Creative Cloud:**
- `~/Library/Application Support/Adobe/` — Adobe application data and plugins
- `~/Library/Caches/Adobe/` — Adobe cache directory
- Typical size: 5-50GB for active users
- Common subdirs: `Common/Media Cache`, `Acrobat/Cache`, `Photoshop/`, `Premiere/`

**Final Cut Pro:**
- `~/Movies/` — Render files and project cache (can be gigabytes)
- `~/Library/Caches/com.apple.FinalCut/` — FCP cache
- `~/Library/Saved Application State/com.apple.FinalCutPro.savedState/` — Session state
- Command to find render files: `find ~/Movies -name "*.fcpbundle" -o -name "*_render*"`

**Logic Pro & GarageBand:**
- `~/Library/Audio/Apple Loops/` — Loop library (large, can be recreated)
- `~/Library/Application Support/GarageBand/` — GarageBand data
- `~/Library/Application Support/Logic/` — Logic Pro data
- `~/Library/Caches/com.apple.Logic/` — Logic cache

**Apple Music & Podcasts:**
- `~/Library/Containers/com.apple.Music/` — Music app container
- `~/Library/Containers/com.apple.podcasts/` — Podcasts app container
- Cache directories may contain downloaded episodes (REVIEW if user cares about local access)

**Blender:**
- `~/Library/Application Support/Blender/` — Blender projects and cache
- `~/Library/Caches/Blender/` — Blender render cache

**DaVinci Resolve & Fusion:**
- `~/Library/Application Support/DaVinciResolve/` — Project data
- Render cache: `~/Movies/` (check for resolve render folders)

## 10. Xcode (SAFE/REVIEW Mixed)

**DerivedData (SAFE):**
- `~/Library/Developer/Xcode/DerivedData/` — Build artifacts, build logs, indexed data
- Safe to delete; Xcode will regenerate on next build
- Typical size: 5-50GB for active developers
- Cleanup: Move contents to Trash (`mv` to `~/.Trash/`), never use `rm -rf`

**Archives (REVIEW):**
- `~/Library/Developer/Xcode/Archives/` — App Store and TestFlight builds
- May contain signed binaries ready for submission
- Delete only if app has been released and no need to maintain build history
- Typical size: 1-5GB per app per build

**CoreSimulator Caches (SAFE):**
- `~/Library/Developer/CoreSimulator/` — iOS Simulator data and caches
- Safe to delete; simulators will be recreated on next run
- Command: `xcrun simctl delete unavailable`
- Typical size: 10-50GB

**Device Support Files (SAFE):**
- `~/Library/Developer/Xcode/iOS DeviceSupport/` — iOS version support files
- Safe to delete; will be re-downloaded on next device connection
- Keep only: Current iOS version being developed for
- Old versions: Safe to remove if not developing for older iOS versions
- Example: Can safely delete `14.0 (18A372)` if only supporting iOS 16+

**Xcode Cache (SAFE):**
- `~/Library/Caches/com.apple.dt.Xcode/` — Xcode build and index caches
- Safe to delete; will be regenerated
- Typical size: 2-10GB

**Xcode Previews (SAFE):**
- `~/Library/Developer/Xcode/UserData/Previews/` — SwiftUI preview builds
- Safe to delete; will be regenerated on next preview refresh
- Typical size: 1-5GB

**Xcode Docsets (REVIEW):**
- `~/Library/Developer/Shared/Documentation/DocSets/` — Downloaded documentation
- Safe to delete but user may need to re-download
- Typical size: 100MB-1GB per docset

## 11. Android Development (SAFE/REVIEW Mixed)

**Android Studio Caches (SAFE):**
- `~/Library/Caches/Google/AndroidStudio*/` — Build cache and indexes
- Safe to delete; will be regenerated
- Typical size: 5-20GB

**Android Studio Logs (SAFE):**
- `~/Library/Logs/Google/AndroidStudio*/` — Application logs
- Safe to delete if >30 days old
- Typical size: 100MB-1GB

**Gradle Caches (SAFE):**
- `~/.gradle/caches/` — Gradle build cache
- `~/.gradle/wrapper/dists/` — Gradle wrapper distributions
- Safe to delete; will be re-downloaded
- Cleanup: Move to Trash (`mv` to `~/.Trash/`), never use `rm -rf`
- Typical size: 2-10GB

**Android SDK Platforms (REVIEW):**
- `~/Library/Android/sdk/platforms/` — Android OS versions (android-29, android-30, etc.)
- Needed for development; delete only if not developing for older Android versions
- Typical size: 500MB-2GB per platform
- Command to list: `ls ~/Library/Android/sdk/platforms/`

**Android System Images (REVIEW):**
- `~/Library/Android/sdk/system-images/` — Emulator system images
- Needed for Android emulator; delete only if not using emulator for those Android versions
- Typical size: 1-4GB per system image

**Android Build Tools (REVIEW):**
- `~/Library/Android/sdk/build-tools/` — Build tool versions (aapt, aapt2, d8, etc.)
- Keep latest version; older versions can be deleted if not maintaining legacy projects
- Typical size: 100-500MB per version
- Cleanup: Move old versions to Trash (e.g., `mv ~/Library/Android/sdk/build-tools/29.0.0 ~/.Trash/`)

**Android NDK (REVIEW):**
- `~/Library/Android/sdk/ndk/` — Native Development Kit for C/C++ development
- Delete only if not doing native Android development
- Typical size: 500MB-2GB

**Android AVD Emulators (REVIEW):**
- `~/.android/avd/` — Android Virtual Device configurations and images
- One folder per emulator; safe to delete if emulator is no longer needed
- Typical size: 1-5GB per AVD
- Command to list: `emulator -list-avds`

**NEVER DELETE - Android Signing Keys:**
- `~/.android/debug.keystore` — Debug signing key (needed for every app build)
- `~/.android/adbkey` — ADB connection key
- `~/.android/adbkey.pub` — ADB public key
- Loss of debug.keystore requires uninstalling the app from all test devices

## 13. Docker (SAFE/REVIEW Mixed)

**Check if Docker Installed:**
```bash
command -v docker
# If output: "not found" → Docker not installed; skip this section
```

**Dangling Images (SAFE):**
- Unused images left over from Docker builds
- Command to list: `docker image ls -f dangling=true`
- Command to clean: `docker image prune` (use `-a` to clean all unused images)
- Typical size: 100MB-10GB total

**Docker Build Cache (SAFE):**
- Build cache from Docker builds
- Command: `docker builder prune`
- Safe to delete; will be recreated
- Typical size: 1-5GB

**Docker System Overview:**
- Get full breakdown: `docker system df -v`
- Shows images, containers, volumes, and build cache with sizes

**Dangling Volumes (REVIEW):**
- Unused Docker volumes (database data, persistent state)
- List dangling: `docker volume ls -f dangling=true`
- **Warning:** May contain database data or important application state
- Only delete if you are certain the data is not needed
- Typical size: 100MB-50GB+ depending on usage

**Unused Networks (SAFE):**
- Unused custom Docker networks
- Command: `docker network prune`
- Safe to delete; will be recreated

**Unused Containers (REVIEW):**
- Stopped containers
- List: `docker container ls -a`
- Delete only containers you no longer need
- May contain configuration you want to preserve

## 14. Stale Project Artifacts (REVIEW)

**node_modules (JavaScript/TypeScript):**
- Located: `find ~ -name "node_modules" -type d -maxdepth 5`
- Safe to delete only if:
  - Parent directory contains `package.json`
  - Parent git repository has no activity <7 days
  - Check: `git log -1 --format=%ct <repo-root>`
- Typical size: 100MB-2GB per project
- Regenerate: `npm install` or `yarn install` or `pnpm install`

**Rust Target (Rust):**
- Located: `find ~ -name "target" -type d -maxdepth 5`
- Safe to delete if:
  - Parent directory contains `Cargo.toml`
  - Project not currently being built
- Typical size: 500MB-10GB per project
- Regenerate: `cargo build`

**Python Virtual Environments:**
- Located: `find ~ -name "venv" -o -name ".venv" -type d -maxdepth 5`
- Safe to delete if dependencies are in `requirements.txt`
- Regenerate: `python3 -m venv venv` + `pip install -r requirements.txt`
- Typical size: 100MB-500MB per venv

**Build Directories:**
- Located: `find ~ -name "build" -type d -maxdepth 4`
- Safe to delete if project has `setup.py`, `CMakeLists.txt`, etc.
- Regenerate: `python setup.py build`, `cmake .`, etc.
- Typical size: 100MB-1GB per project

**Python __pycache__:**
- Located: `find ~ -name "__pycache__" -type d`
- Safe to delete; automatically regenerated when Python runs
- Typical size: 10MB-500MB total
- No regeneration needed; auto-created

## 15. Spotlight Index (SAFE)

**Spotlight Database:**
- Location: `/.Spotlight-V100/`
- Safe to delete; will be rebuilt automatically
- Typical size: 1-5GB

**Rebuild Command:**
- `sudo mdutil -E /` (rebuilds for entire volume)
- Rebuild time: 5-30 minutes depending on disk size
- Disable/enable: `mdutil -a on` or `mdutil -a off`

## 16. Size Estimation Commands

**Get Size of Directory:**
```bash
du -sh <path>
# Example: du -sh ~/Downloads/
# Output: 23.4G     ~/Downloads/
```

**Find Large Files:**
```bash
mdfind "kMDItemFSSize > 524288000"
# Finds all files >500MB on the system
```

**Find Large Files in Specific Location:**
```bash
find ~ -size +1G -type f
# Finds all files >1GB in home directory
```

**Top Directories by Size:**
```bash
du -sh ~/* ~/Library/* 2>/dev/null | sort -rh | head -20
# Shows top 20 largest directories in home and Library
```

**Disk Usage Overview:**
```bash
df -h /
# Shows total disk space, used, available
```

**List All Files with Sizes:**
```bash
du -sh <directory>/* | sort -rh
# Lists all items in directory sorted by size
```

**Find Files Modified Before Date:**
```bash
find ~ -mtime +30 -type f
# Finds files not modified in last 30 days
```

**Find Files Modified After Date:**
```bash
find ~ -mtime -7 -type f
# Finds files modified in last 7 days
```

**Estimate Time Machine Snapshots:**
```bash
tmutil listlocalsnapshots / | wc -l
# Count of local snapshots
```

**Docker System Analysis:**
```bash
docker system df -v
# Detailed breakdown of Docker disk usage
```
